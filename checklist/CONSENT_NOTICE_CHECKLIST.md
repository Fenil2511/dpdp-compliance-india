# DPDP Consent Notice Review Checklist
## Section 5, DPDP Act 2023 + Rule 3, DPDP Rules 2025

> **Use this checklist to**:
> (a) Review your existing consent notice/privacy policy for DPDP compliance
> (b) Draft a new consent notice that meets all Rule 3 requirements
> (c) Audit a client's or vendor's consent notice

---

## PART A — MANDATORY CONTENT CHECKLIST (Rule 3)

Rate each item: ✅ Present & Adequate | ⚠️ Present but Inadequate | ❌ Missing

### Element 1 — Identity of Data Fiduciary

| # | Check | Status | Notes |
|---|-------|--------|-------|
| 1.1 | Legal name of the Data Fiduciary is clearly stated | | |
| 1.2 | Registered address is included | | |
| 1.3 | Website URL is included | | |
| 1.4 | Contact email or phone for privacy queries is included | | |

### Element 2 — Itemised Personal Data List (Rule 3 Core Requirement)

| # | Check | Status | Notes |
|---|-------|--------|-------|
| 2.1 | Each category of personal data is listed **separately** (not bundled as "your data" or "personal information") | | |
| 2.2 | Data types are **specific** (e.g., "your full name" not "identity data") | | |
| 2.3 | ALL data categories actually collected are listed (no hidden collection) | | |
| 2.4 | Sensitive data types (health, financial, biometric) are explicitly named | | |
| 2.5 | Device/behavioural data collected is listed (if applicable) | | |
| 2.6 | Cookies and tracking data listed (if applicable) | | |

---

### Element 3 — Purpose for Each Data Element (Rule 3 Core Requirement)

| # | Check | Status | Notes |
|---|-------|--------|-------|
| 3.1 | A specific purpose is stated for **each** data type (not one blanket purpose for all) | | |
| 3.2 | Purposes are **specific and granular** (not vague like "improve our services" or "provide better experience") | | |
| 3.3 | Purposes are limited to what is **necessary** — no "we may use your data for future products" type language | | |
| 3.4 | Purposes are **lawful** (not expressly forbidden by law) | | |
| 3.5 | If a legitimate use (Section 7) is the basis (not consent), this is stated | | |

### Element 4 — Third-Party Sharing Disclosure

| # | Check | Status | Notes |
|---|-------|--------|-------|
| 4.1 | All third parties who receive personal data are named or sufficiently described | | |
| 4.2 | The **purpose of sharing** with each third party is stated | | |
| 4.3 | Whether third parties are in India or overseas is indicated | | |
| 4.4 | Explicit statement that personal data is **not sold** to third parties | | |
| 4.5 | Categories of Data Processors engaged are disclosed | | |

### Element 5 — Data Retention

| # | Check | Status | Notes |
|---|-------|--------|-------|
| 5.1 | Retention period is stated for each data category (or criteria used to determine retention) | | |
| 5.2 | Retention periods are **specific** (not just "as long as necessary") | | |
| 5.3 | Where legal retention obligation applies (RBI, SEBI, PMLA, etc.), this is mentioned | | |
| 5.4 | What happens to data after retention period (deletion method) is described | | |

| # | Check | Status | Notes |
|---|-------|--------|-------|

| 6.2 | **Right to Correction** (Section 12) is mentioned with process to exercise | | |

| 6.4 | **Right to Grievance** (Section 13) is mentioned | | |

### Element 7 — Consent Withdrawal Mechanism (Rule 4)

## PART B — FORMAT AND LANGUAGE COMPLIANCE

| # | Check | Status | Notes |
|---|-------|--------|-------|
| B.1 | Notice is available in **English** | | |
| B.2 | Notice is available in at least **one Eighth Schedule language** relevant to user base | | |
| B.3 | User is given the **option to choose their language** | | |
| B.4 | Notice is written in **plain, simple language** (not legalese) | | |
| B.5 | Notice is **concise** — no irrelevant information stuffed in | | |
| B.6 | Notice is **NOT buried** in 40+ page Terms of Service | | |
| B.7 | Notice is given **before or alongside** the consent request (not after) | | |
| B.8 | If delivered digitally: notice is **readable on mobile** | | |
| B.9 | Layered format used: **summary** first, detail available on click/tap | | |

## PART C — CONSENT CAPTURE COMPLIANCE

| # | Check | Status | Notes |
|---|-------|--------|-------|
| C.1 | Consent is obtained through a **clear affirmative action** (tick-box, button) | | |
| C.2 | Consent checkbox/button is **NOT pre-ticked** | | |
| C.3 | Consent is **free** — service is not denied for withdrawing optional consents | | |
| C.4 | Consent is **specific** — separate checkboxes for separate purposes (e.g., marketing separate from core service) | | |
| C.5 | Consent for **children** requires parent/guardian action (if applicable) | | |
| C.6 | Consent record is **stored** (what was consented, when, by whom) | | |
| C.7 | If pre-Act consent is being relied upon — **retroactive notice** has been given | | |

## PART D — COMMON FAILURES (RED FLAGS)

If any of these are present, the notice is likely NON-COMPLIANT:

| Red Flag | Problem |
|----------|---------|
| "By using our service, you agree to our privacy policy" | Not a valid affirmative action for consent |
| "We may use your data to improve our products and services" | Not a specific purpose |
| "We may share your data with our partners" | Third parties must be named or specifically described |
| "We retain your data as long as necessary" | Not a specific retention period |
| "To exercise your rights, contact us" | Not actionable — must specify how |
| Notice only available in English | Must offer at least one Eighth Schedule language |
| No mention of Data Protection Board | Mandatory per Rule 3 |
| Notice inside Terms of Service | Must be separate / clearly highlighted |
| Pre-ticked marketing consent | Invalid consent; must be opt-in |
| No grievance officer contact | Mandatory per Section 13 |

## PRIORITY FIX ORDER

If you have fails, fix in this order (highest penalty exposure first):

1. **Add Board complaint reference** (Rule 3 mandatory — ₹50 Cr)
2. **Itemise personal data and purposes** (Rule 3 core — ₹50 Cr)
3. **Add withdrawal mechanism** (Rule 4 — ₹50 Cr)
4. **Add language option** (Eighth Schedule — ₹50 Cr)
5. **Add Grievance Officer with contact** (Section 13 — ₹10,000/instance)
6. **Make consent affirmative** (not pre-ticked — ₹50 Cr)
7. **Separate from T&Cs** (Rule 3 — ₹50 Cr)
8. **Add rights exercise process** (Sections 11–14 — ₹10,000/instance)
