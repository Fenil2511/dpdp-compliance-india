# Contributing to DPDP Compliance India

Thank you for your interest in contributing! This repository aims to be
the most comprehensive and accurate DPDP compliance resource available.

---

## 🎯 What We Need

### High Priority Contributions
- **Regulatory updates**: When MeitY notifies SDFs, restricted countries, or
  exemptions — update the relevant sections
- **Board Orders**: When DPBI issues first orders/penalties — add case summaries
- **Sector guidance**: Telecom, insurance, government, retail, logistics sectors
- **State-specific guidance**: Maharashtra, Tamil Nadu, Karnataka have active
  tech ecosystems — sector-specific state guidance welcome
- **Indian language translations**: Translate key templates to Hindi, Tamil,
  Gujarati, Telugu, etc. for broader accessibility
- **GRC tool integrations**: Integration guides for OneTrust, ServiceNow,
  MetaCompliance, or Indian GRC tools

### Medium Priority
- Additional worked examples (startups, SaaS companies, manufacturing)
- DPO job description templates
- Data retention schedule templates by sector
- Training materials / quiz content

---

## 📋 Contribution Process

1. **Fork** the repository
2. **Create a branch**: `git checkout -b feature/your-contribution`
3. **Make changes**: Follow the style guide below
4. **Verify**: Every claim must be traceable to the Act, Rules, or official guidance
5. **Pull Request**: Open a PR with a clear description of what you changed and why

---

## 📝 Style Guide

### Accuracy First
- **Cite the section**: Every compliance requirement must reference the specific
  Act section or Rule number (e.g., "Section 8(5)" or "Rule 6")
- **No unsupported claims**: Do not invent requirements that are not in the Act or Rules
- **Date your sources**: If citing Board orders or guidance, include the date

### Format
- Use Markdown tables for comparisons
- Use checkboxes for checklists
- Use code blocks for templates
- Use the priority emoji system: 🔴 Critical | 🟠 High | 🟡 Medium | 🟢 Low

### Language
- Plain English — avoid legal jargon
- Business-friendly — compliance advice should be actionable
- India-specific — do not copy from GDPR guides without India-specific adaptation

---

## ⚠️ What We Don't Accept

- **Vague guidance** without specific Act/Rule citations
- **GDPR transplants** — India's DPDP has significant differences; don't copy GDPR templates
- **Legal advice** — contributions should be educational, not jurisdiction-specific legal advice
- **Marketing content** for specific vendors or tools
- **Inaccurate regulatory claims** — any inaccuracy will be flagged and corrected

---

## 🐛 Reporting Errors

If you find an error (incorrect section reference, outdated information, etc.):
1. Open an **Issue** with the label "accuracy"
2. Describe what is incorrect and what the correct information is
3. Reference the official source (Act, Rules, MeitY gazette)
